SonarLint for Command Line

SonarLint for Command Line provides pre-commit feedback to developers on new bugs and quality issues injected into code from any language supported by the SonarQube ecosystem.

SonarLint offers the ability to scan code for issues before checking it in. After installing the plugin on the server, and a SonarQube Scanner locally, you just have to run an analysis.

SonarLint for Command Line is free, open source, and available in the SonarQube Update Center.

Requirements

The tree structure of the source code on your local machine must match the tree structure that has been remotely analyzed by SonarQube

Installation

  1. Install your favorite scanner (SonarQube Scanner , SonarQube Scanner for Maven or SonarQube Scanner for Ant) on your local machine, and set the sonar.host.url property to point to your remote SonarQube server.

Usage

  1. Copy the configuration file (sonar-project.properties, pom.xml, etc.) used to analyze the project on the remote server to your local machine.
  2. Adjust the values:
    • sonar.sources - must refer to the directory containing the source code on your local machine.
    • sonar.issuesReport.html.enable=true - to get reports in HTML format
    • sonar.issuesReport.html.location - to define the output location of the HTML report. May be relative or absolute. The default value is .sonar/issues-report/ for the SonarQube Runner and SonarQube Runner for Ant, and target/sonar/issues-report/ for SonarQube Runner for Maven.
    • sonar.issuesReport.lightModeOnly=true - to skip full report generation. This may be useful when working on legacy projects with a lot of issues
    • sonar.issuesReport.console.enable=true - to display a short summary in the console:
  3. Run an analysis:
    sonar-runner -Dsonar.analysis.mode=incremental -Dsonar.issuesReport.html.enable=true
    or to see all issues:
    sonar-runner -Dsonar.analysis.mode=preview -Dsonar.issuesReport.html.enable=true
