SonarLint for Eclipse

SonarLint is an Eclipse plugin that provides on-the-fly feedback to developers on new bugs and quality issues injected into Java, JavaScript and PHP code.

SonarLint offers a fully-integrated user experience in Eclipse-based IDEs. After installing the plugin you just have to enable SonarLint on your projects and issues will be reported as Eclipse markers.

SonarLint is free, open source, and available in the Eclipse Marketplace.

FAQ

  • How to enable SonarLint on my project? - Right click on the project -> Configure -> Enable SonarLint
  • How to start SonarLint analysis? - SonarLint analysis is automatically triggered by a builder (in short: when you save your file). You can also run a full analysis of your project: Right click on the project -> SonarLint -> Analyze all files
  • How to configure rules? - You can't for now. We will make this feature available in upcoming versions, probably by requiring to "link" SonarLint with an existing SonarQube server instance.
  • How to exclude files from the analysis? - By default we are excluding derived files. Appart from that all Java, JavaScript and PHP files are analyzed. Advanced users can configure some properties (either in global or per project SonarLint configuration) like sonar.exclusions.
  • How to update SonarLint? - SonarLint Eclipse Plugin can be updated like any other Eclipse plugin. During the first analysis, it will also download rule engines from https://update.sonarlint.org. Updates of the rule engines will be published on a regular basis. To download latest rule engines use Help -> Update SonarLint menu.
